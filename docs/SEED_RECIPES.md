# SEED_RECIPES.md — 朝書く5本のシードレシピテンプレ

このドキュメントは Tan-i プラットフォームを立ち上げるために、
**創業者（あなた）が自分の経験から書く 5 本のシードレシピ**のテンプレ。

実装より先に、これを書く。
書けるかどうか、書いたら他人に役立ちそうか、自分が書きたいかを自分で判断する。

---

## なぜ最初に書くのか

- コードを書く前に「**自分が本当に書けるか**」を確かめる
- 書いてみて読み返した時に「これは他人に価値があるか」を体感する
- 5本書ければ MVP の価値は確認できる
- 書けない / つまらない → ピボット先を再検討

## レシピ 2 〜 5

レシピ1と同じテンプレで4本書く。

科目を選ぶコツ：

- **対照的なペア**：難しかった科目 × 楽だった科目（読者が選びやすい）
- **取得方法の違い**：レポート系 × 試験系 × スクーリング系
- **失敗例 1 本入れる**：成功だけでは信頼が薄い、失敗→再挑戦が一番刺さる
- **必修科目を 1 本**：多くの人が探しに来る検索キーワード

### おすすめのラインナップ例

1. 楽だった科目（★1〜2）
2. 難しかった科目（★4〜5）
3. レポート中心の科目
4. 試験一発の科目
5. **失敗→再挑戦した科目** ← これが一番刺さる

## 書き終わったら自問

5本書き終わって、声に出して読んでみる。

- [ ] 自分が読んでも面白いか
- [ ] 他の人が読んだら、再現できそうか
- [ ] 似た立場の後輩に「これ読んで」と渡せる質か
- [ ] 5本書く中で、書きたいネタがまだ思いついたか
- [ ] 全部書くのに何時間かかったか

これに「Yes」が3つ以上 → MVP 実装着手。
「No」が多い → 設計か、対象市場か、自分の関与度か、何かを見直す。
