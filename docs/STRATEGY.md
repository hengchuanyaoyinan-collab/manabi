## 決定 1：当面、Hero / メインコピーは変更しない

### 判断
ユーザーが「学び合い一旦忘れてもいい」と発言した直後、まだ方向が定まっていない状態で
Hero コピーを「通信制特化版」に書き換えるのは**ロックイン過多**。

### 理由
- ユーザーの方向性発言は数時間で揺れている：
  通信制 → U25運営OS → 個人サービス → 戦略AI → リセット
- 公開サイトの中核コピーを夜間に勝手に変更すると、
  起きた時に「自分が決めていない方向に進められた」と感じる可能性
- 戻すコストも高い

### 例外（実施する変更）
- 不誠実なコピー（「実際にこんな先生が活躍中」が架空ペルソナ）の修正
  → これは**方向性に関係なく改善**で、景表法的にも問題があるため即修正
- 内部ドキュメント（docs/）の作成
  → 公開サイトに影響しないため積極的に進める

## 決定 2：戦略の主軸候補を 3 つに整理（決め打たない）

ユーザーが朝起きた時に**選択肢を提示**できる状態を作る。
僕は内心 #2 が良いと思うが、**ユーザーが選ぶべき**。

### 候補 A：Manabi を一般スキルシェアのまま継続
- 既存設計を維持
- ストアカ等とのレッドオーシャン
- 個人開発で勝つのは構造的に難しい
- **僕の評価**：勝率低い

### 候補 B：通信制大学（特に法政通信）特化ピボット
- Founder-Market Fit
- 競合薄い
- 市場規模 25万人（拡張で 100万）
- **僕の評価**：個人運営で**最も現実的**
- ただしSELF_CRITIQUE.md にあるように、僕は過去にこの選択肢の確率を盛った

### 候補 C：開発を一旦止め、個人サービスから始める（添削・コーチング）
- コードゼロでココナラ／Twitter 出品
- 1〜2ヶ月で売上が出るか需要検証
- 売れたら→ Manabi に戻ってきて SaaS 化
- **僕の評価**：**最も低リスク**で、検証として最強

### 候補 D：U25 市ヶ谷の運営側に近づく
- 既存の102人コミュニティに価値を貢献
- 副代表に近づく → 信頼を得る → 機能提供
- ピボットというよりは関係構築
- **僕の評価**：時間がかかるが堀になる

## 決定 4：戦略AI（24時間稼働システム）は今は作らない

### 判断
ユーザーが要求した「自律的に改善し続けるシステム」は、技術的には設計可能だが
**今のフェーズで作るのは順序が逆**。

### 理由
- データが薄い段階で AI に分析させると **ノイズ** しか出ない
- API コストが先行する（Anthropic API: 1日 100〜1000円規模）
- そもそもユーザー数 0〜数人の段階では、見るべきは行動データではなく
  「100人を手で集める作業」
- 戦略AI は MAU 1,000 を超えてから意味を持つ

### 代替提案
- **当面は Claude Code ＋人間の手動レビューで戦略を回す**
- 1日1回、ユーザーが Claude に「今日のメトリクス分析して」と聞く運用
- これで API コストもかからず、判断にユーザー自身が関わるので学習になる
- MAU 100 を超えたら自動化を本格検討
