# 🔧 動画制作自動化フロー (現行 v4 パイプライン)

**ブンブンの政治権文録** で毎日 1 本を AI 自動で投稿するためのフロー全体図。

## 全体

```
┌──────────────────────────────────────────────────────────┐
│ 1. 題材選定 (毎日 18:30 起動)                             │
│    - config/topic-queue.json 先頭を pop                  │
│    - insights.json から競合トレンドも考慮                  │
└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 2. 台本生成                                                │
│    - claude CLI で v4 prompt 実行                         │
│    - 48〜60 シーン、感情タグ付き                           │

└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 3. 音声合成                                                │
│    - VOICEVOX (localhost:50021) または ElevenLabs         │
│    - 各シーンごとに wav 保存                               │

└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐

               ↓
┌──────────────────────────────────────────────────────────┐

               ↓

│    - 24 fps (本番) / 30 fps (ハイライト)                  │
│    - シーン MP4 を concat で結合                           │
└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 7. サムネ生成                                              │
│    - 極太黄/赤テキスト + キャラ + 肖像背景                  │
│    - A/B 2 案生成 (将来)                                  │
└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 8. メタデータ生成                                          │
│    - タイトル (台本データから)                             │
│    - 概要欄 (章タイムスタンプ付き)                         │
│    - タグ 15 個                                           │
│    - 公開時刻 (19:00 JST)                                 │
└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 9. YouTube アップロード                                    │
│    - YouTube Data API v3 (OAuth)                          │
│    - resumable upload                                     │
│    - サムネ設定                                            │
│    - 公開予約                                              │
│    - プレイリスト追加                                      │
└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 10. 記録                                                  │
│     - config/analytics.json に記録開始                   │
│     - topic-queue.json の posted に移動                   │
│     - Slack/Discord 通知 (将来)                           │
└──────────────┬───────────────────────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│ 11. 翌朝分析 (毎日 09:00)                                 │
│     - 前日動画の再生数・CTR を取得                         │

│     - 改善案を insights.json に追記                        │

```

## 時間配分 (1 本あたり)

| ステップ | 時間 | ボトルネック |
|---|---|---|
| 2. 台本生成 | 3〜5 分 | claude CLI レート |
| 3. 音声合成 | 1〜2 分 | VOICEVOX 処理 |
| 4. 画像収集 | 30 秒〜2 分 | ネットワーク |
| 5-6. 動画組立 | 8〜15 分 | CPU (並列可) |
| 7. サムネ | 5 秒 | - |
| 8-9. 投稿 | 1〜3 分 | アップロード速度 |
| **合計** | **15〜30 分** | - |

**1 日 1 本なら余裕で回る。2 本以上も物理的に可能。**

## 実行コマンド (現行)

```powershell
.\scripts\setup-windows-task.ps1
```

## 冗長化・エラー処理

### 各ステージで失敗したら
1. `report.json` に stage=failed を記録
2. 緊急時はフォールバック起動:
   - 台本生成失敗 → ストックから選ぶ (test_data/*.json)
   - 音声合成失敗 → open-jtalk 自動切替
   - 画像取得失敗 → パステルグラデ
   - アップロード失敗 → 翌朝リトライ

### 失敗通知 (未実装、Phase 1 で追加)
- Discord Webhook で「投稿失敗」通知
- ログは `logs/YYYY-MM-DD.log` に全保存
